## Summary

Adds a `--tdd` flag to `/nob implement` that inserts a `nob:test-writer` agent between Tech Lead and Dev, enforcing Red-Green-Refactor. The feature is already largely implemented across the core skills; this design doc captures the remaining gaps and the fixes needed.

---

## Current Implementation State

The following is already fully implemented and requires no changes:

- **`skills/nob/SKILL.md`**: `--tdd` flag parsed, `TDD_FLAG` and `TEST_WRITER_MODEL_RESOLVED` extracted, quick path unsupported message, lite path full TDD inline dispatch (test-writer dispatch, prompt, `LITE_TDD_STATUS`), full path `TDD flag:` forwarded in [INPUTS], `TDD:` line in terminal summary, Output Block Validation table includes Test Writer.
- **`skills/nob/path-full/SKILL.md`**: `TDD_FLAG`, `TEST_WRITER_MODEL`, `TDD_ACTIVE`, `TDD_STATUS` variables; Tech Lead dispatch includes `TDD flag:`; Reviewer dispatch includes `TDD flag:` and `TDD test files:`; `TDD status:` in `[FULL PATH OUTPUT]`.
- **`skills/nob/path-lite/SKILL.md`**: `TDD_FLAG` and `TDD_TEST_FILES` setup; Dev and Reviewer dispatches receive TDD inputs.
- **`skills/nob/path-quick/SKILL.md`**: No changes needed; hub prints unsupported message before quick path runs.
- **`skills/test-writer/SKILL.md`**: Fully implemented.
- **`skills/tech-lead/SKILL.md`**: Step 2.7 dispatches test-writer, Step 4.5 sets `TDD_STATUS`, Step 3 passes `TDD mode:` and `TDD test files:` to dev, output format forwards `[TEST WRITER OUTPUT]`.
- **`skills/dev/SKILL.md`**: Reads `TDD mode:` and `TDD test files:`, runs TDD pre-run check (Red phase).
- **`skills/reviewer/SKILL.md`**: Reads `TDD flag:` and `TDD test files:`, Step 3.4 TDD pass check.

## Interfaces / Contracts

### `[TEST WRITER OUTPUT]` block

- **Produced by:** `skills/test-writer/SKILL.md`
- **Consumed by:** `skills/tech-lead/SKILL.md` (Step 2.7), `skills/nob/SKILL.md` (lite path inline), `skills/nob/path-full/SKILL.md` (Phase 3 Reviewer dispatch)
- **Required fields:** `Units tested:`, `Test files written:`, `Tests written:`, `Framework detected:`
- **Optional fields:** `Acceptance criteria covered:`, `Items not tested (needs human):`

### TDD inputs passed to Dev

```
TDD mode: {true|false}
TDD test files: {comma-separated paths, or: none}
```

### TDD inputs passed to Reviewer

```
TDD flag: {true|false}
TDD test files: {comma-separated paths, or: none}
```

### t1 — Add Test Writer to Output Block Validation in path-full

**File:** `skills/nob/path-full/SKILL.md`

| Test Writer | `Units tested:`, `Test files written:`, `Tests written:`, `Framework detected:` |
```

### t2 — Add Test Writer to Output Block Validation in path-lite

Add to the required fields table:
```
| Test Writer | `Units tested:`, `Test files written:`, `Tests written:`, `Framework detected:` |

### t3 — Add `TDD_TEST_FILES` extraction in path-full after Tech Lead

**File:** `skills/nob/path-full/SKILL.md`  
**What's missing:** After extracting `[TEST WRITER OUTPUT]` from Tech Lead's response (line 231), `TDD_TEST_FILES` is referenced later in the Reviewer dispatch but is never explicitly populated. The variable needs to be assigned by extracting paths from `TEST_WRITER_OUTPUT "Test files written:"`. Also set `TDD_ACTIVE = true` when TEST_WRITER_OUTPUT is not "none".

After the line `Extract [TEST WRITER OUTPUT]... Store as TEST_WRITER_OUTPUT`, add:
```
If TEST_WRITER_OUTPUT is not "none": set TDD_ACTIVE = true. Extract TDD_TEST_FILES = comma-separated paths from the "Test files written:" section of TEST_WRITER_OUTPUT.
```

### t4 — Add `TDD status:` field to `[LITE PATH OUTPUT]` in path-lite

**File:** `skills/nob/path-lite/SKILL.md`  
**What's missing:** Step 7 `[LITE PATH OUTPUT]` does not include a `TDD status:` field. While the hub computes `LITE_TDD_STATUS` inline for its own terminal summary, path-lite should also emit this field so it is self-contained and consistent when used standalone.

Add `TDD status:` line to `[LITE PATH OUTPUT]`:
```
TDD status: {if TDD_FLAG = true and DEV/Reviewer PASS: "Red ✓ → Green ✓"; if TDD_FLAG = true and fail: "Red ✓ → Green ✗"; else: "skipped"}
```

Also update `Agents run:` line to conditionally include test-writer:
```
Agents run: {if TDD_FLAG: "test-writer({test-writer model}) · "}pm({pm model}) · dev({dev model}) · reviewer({reviewer model})
```

### t5 — Add `TDD flag:` and `test-writer` model to retry's Tech Lead dispatch

**File:** `skills/nob/retry/SKILL.md`  
**What's missing:** Step 9 (Tech Lead retry dispatch) `[INPUTS]` block is missing `TDD flag: false` and `test-writer` in the Agent models section. On a retry run TDD should not re-dispatch the test-writer (tests already exist), so TDD flag should be explicitly set to false.

In the Step 9 Tech Lead retry dispatch `[INPUTS]`:
1. Add `test-writer: {test-writer model from [INPUTS], or: haiku}` to the Agent models section.
2. Add `TDD flag: false` — retry never re-dispatches the test-writer.
3. Add `Agents enabled: {Agents enabled from [INPUTS], comma-separated}` so Tech Lead can check if designer is enabled.
